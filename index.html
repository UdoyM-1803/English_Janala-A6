Frequently Asked Questions

1. What is the difference between var, let and const?
var is function-scoped, hoisted, and can be redeclared, which may cause bugs. let and const are block-scoped and safer. let allows reassignment, while const does not. Use const by default, and let when values need to change. Avoid var in modern JavaScript development.
2. What is the difference between map(), forEach(), and filter()?
map() creates a new array by transforming each element. filter() creates a new array with elements that satisfy a condition. forEach() simply iterates over elements without returning anything. Use map and filter when you need a result array; use forEach for side effects.
3. What are arrow functions, and how are they different from regular functions?
Arrow functions provide a shorter syntax and inherit this from their surrounding scope, making them useful in callbacks. Regular functions have their own this, can be used as constructors, and support arguments. Arrow functions are not suitable when dynamic context or object methods requiring this are needed.
4. How do javascript promises work?
Promises manage asynchronous operations and avoid callback hell. They have three states: pending, fulfilled, and rejected. You can chain .then() for success and .catch() for errors. .finally() runs regardless of outcome. Promises improve readability and are often used with async/await for cleaner asynchronous code.
5. How does closure work in JavaScript?
A closure is created when a function remembers variables from its outer scope even after that outer function has finished executing. It allows data encapsulation and private variables. Closures are commonly used in callbacks, event handlers, and factory functions to maintain state across function calls.
